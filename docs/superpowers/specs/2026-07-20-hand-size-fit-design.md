# SigTouch v1.7 设计文档:手影尺寸钳制与出界修复

日期:2026-07-20
状态:已确认
关联:v1 §4.2(手部轮廓缩放)、v1.2 §4(光标锚定食指)、v1.4 §2(缩放模型升级)

## 1. 缺陷与调查结论

**现象**:手影在屏幕上过大,并超出屏幕边界(被裁切)。

### 1.1 根因 A(主因):手影占屏比例与屏幕物理尺寸成反比

现有模型 `scale = (d_screen/0.6) × (24/diag) × mult`,几何推导手影占屏比例:

```
手在画面中的归一化宽 = 手物理宽 / 摄像头该距离视野宽 = 0.20 / (1.155·d) = 0.173/d
手影占屏宽 = (0.173/d) × scale = (0.173/d) × (d/0.6) × (24/diag) = 0.288 × 24/diag
```

距离项 `d` 精确抵消(这部分设计正确:人前后移动时手影稳定),但**占屏比例与屏幕对角线成反比**。实测推导值:

| 屏幕 | 手影占屏宽 |
|---|---|
| 13 吋 | 53.3% |
| 16 吋 | 43.3% |
| 24 吋 | 28.9% |
| 65 吋 | 10.7% |

高度方向更严重:摄像头 4:3 画面下 `归一化高 = 0.25/(0.866·d) = 0.289/d`,`diag=24` 时手影**占屏高 48%**。

用户实际配置(笔记本 13–16 吋、设置保持默认 24 吋)对应约 29% 宽 / 48% 高——目标 25% 屏高的近 2 倍。

### 1.2 根因 B(放大):锚点位置导致边缘必然出界

食指尖位于手部 bbox 的**顶部 5%**,手主体向下延伸 95%。`align_to_cursor` 把食指尖钉到光标后,实测(scale=1.5,1920×1080):

| 光标位置 | 出界 |
|---|---|
| 屏幕中心 | 无 |
| 底部中央 | 下 284px |
| 右下角 | 右 209px + 下 284px |
| 左上角 | 左 152px |

**无条件成立**,且手影越大出界越多。B 是 A 的放大,修 A 可大幅缓解 B。

## 2. 修复决策

| 决策 | 结论 |
|---|---|
| 手影目标尺寸 | 手影 bbox 高度 ≤ 屏幕高度的可配置比例,默认 **0.25** |
| 收缩中心 | **围绕食指尖**——保持 v1.2「光标始终在食指上」契约不被破坏 |
| 出界处理 | 尺寸钳制后再做边缘收缩(同样围绕食指尖),使手影落入屏幕;收缩系数设**下限 0.5**,避免光标贴角时手影塌缩为不可见 |
| 物理模型 | **保留**(距离变化时手影稳定是好特性),钳制作为其上限 |
| 兼容 | `overlay_scale` 不改;钳制在 overlay 渲染层,新增纯函数 |

**已知取舍**:光标位于屏幕极端角落且收缩已达下限 0.5 时,手掌仍可能小幅出界。此时食指尖(操作焦点)始终可见,不影响功能。

## 3. 实现设计

### 3.1 新纯函数(`sigtouch/ui/overlay.py`)

```python
def fit_hand_to_screen(points, anchor_idx, screen_w, screen_h,
                       max_h_fraction, min_shrink=0.5):
    """围绕 anchor(食指尖)收缩点集,使其
       (a) bbox 高度 ≤ screen_h × max_h_fraction;
       (b) 尽量落入屏幕矩形内(收缩系数不低于 min_shrink)。
    返回收缩后的点集;anchor 点位置恒不变(保证光标对齐)。
    """
```

- 收缩系数 `k = clamp(min(k_size, k_bounds), min_shrink, 1.0)`;
  - `k_size = (screen_h × max_h_fraction) / bbox_h`(bbox_h ≤ 限制时为 1.0);
  - `k_bounds`:对每个点,令 anchor + k·offset 落在 `[0,W]×[0,H]`,取所有约束的最小上界;
- 点变换:`p' = anchor + k × (p - anchor)`;
- 退化保护:`bbox_h <= 0` 或点数不足直接返回原点集。

### 3.2 渲染管线接入(`OverlayWindow.paintEvent`)

现顺序:`scaled_points → align_to_cursor → silhouette_path`。
新顺序:`scaled_points → align_to_cursor → **fit_hand_to_screen** → silhouette_path`。

`palm_px`(描边宽度基准)在 fit 之后计算,故描边随手影同步收缩,比例不失真。

`cursor_px` 为 None 时(旧行为/无光标)以手部 bbox 中心为 anchor 做尺寸钳制,不做边缘收缩。

### 3.3 配置

`config.DEFAULTS` 新增:

```python
"display/hand_max_screen_fraction": 0.25,  # 手影高度上限(占屏幕高度比例)
```

设置-显示页新增滑杆「手影最大高度」10–60(%) ↔ /100,说明「手影高度上限,占屏幕高度的比例」;轻量键(即时生效,不属于 `_RESTART_KEYS`)。

## 4. 测试策略

- `fit_hand_to_screen`:
  - 尺寸钳制:超高点集被收缩到恰好 `max_h_fraction`;未超高的点集原样返回;
  - anchor 不变量:任意输入下 `out[anchor_idx] == points[anchor_idx]`;
  - 边缘收缩:光标在底部/角落时,收缩后 bbox 落入屏幕(除非触及 `min_shrink` 下限);
  - 下限保护:极端角落时 `k` 不低于 `min_shrink`,手影不塌缩;
  - 退化输入:零高度 bbox、单点集合不抛异常。
- 回归:用户实际配置(1920×1080、scale 对应 48% 屏高)经 fit 后 ≤ 25%;`align_to_cursor` 既有测试不变。
- 设置:新键即时生效、不触发视觉线程重启。
- manual-qa 第 17 项:笔记本上手影约占屏高 1/4;光标移到四角与底边,手影完整或仅极端角落小幅裁剪,食指尖始终贴合光标;调大「手影最大高度」滑杆手影随之变大。

## 5. 明确不做(v1.7)

- 改动 `overlay_scale` 物理模型(距离补偿正确,仅需上限)。
- 为避免出界而移动手影(会破坏「光标始终在食指上」契约)。
- 按手在画面中的真实像素尺寸自适应(当前几何模型已足够)。
