# SigTouch 设计文档

日期:2026-07-16
状态:已确认

## 1. 概述

SigTouch 是一个跨平台(Windows / macOS / Linux)桌面应用,通过普通摄像头识别用户的手部与眼睛,将手以 Oculus 风格的半透明轮廓投影到屏幕上,并用手势完成鼠标(移动、左键、右键、拖拽、滚动)与键盘(回车、退格)操作。应用常驻后台,提供系统托盘图标与设置窗口,以独立安装包形式分发。

**眼睛识别的用途**:估算用户与屏幕的距离,用于自适应缩放屏幕上手部轮廓的渲染大小——屏幕越大或人越远(如电视、幕墙场景),手部轮廓渲染得越大,保证用户始终看得清。眼睛不用于光标定位;光标完全由手的位置驱动。

**许可决策**:项目保持 MIT 协议,因此 GUI 采用 **PySide6(LGPLv3,Qt 官方 Python 绑定)** 而非 PyQt6(GPLv3)。其余核心依赖 MediaPipe(Apache 2.0)、OpenCV(Apache 2.0)与 MIT 兼容;pynput(LGPL)动态使用无冲突。

## 2. 技术选型

| 用途 | 选型 | 理由 |
|---|---|---|
| 手部/人脸识别 | MediaPipe Tasks(Hand Landmarker + Face Landmarker) | 谷歌官方、CPU 实时、三平台支持、可随 PyInstaller 打包 |
| 摄像头采集 | OpenCV `VideoCapture` | 三平台统一 API,设备枚举与重连成熟 |
| GUI / 托盘 / 覆盖层 | PySide6 | 一站式解决托盘、设置窗、以及跨平台透明点击穿透覆盖层;LGPL 兼容 MIT |
| 输入注入 | pynput | 鼠标移动/点击/滚动与键盘按键,三平台支持 |
| 配置持久化 | QSettings | 自动落到各平台原生位置 |
| 打包 | PyInstaller(onedir) | onedir 模式下 Qt 动态库独立存在,满足 LGPL;三平台产物成熟 |

架构形态:**单进程**。视觉管线在独立线程运行,通过 Qt 信号与主线程通信;不采用多进程(第一版崩溃隔离需求由看门狗满足,避免 IPC 复杂度与延迟)。

### 模块划分

| 模块 | 职责 | 依赖 |
|---|---|---|
| `sigtouch/capture` | 摄像头采集线程:设备枚举、分辨率/帧率配置、断线重连 | OpenCV |
| `sigtouch/perception` | 手部 21 关键点、左右手、人脸虹膜、人-屏距离估计;输出统一 `FrameResult` 数据类 | MediaPipe |
| `sigtouch/interaction` | 光标映射(交互框→屏幕、One Euro 滤波)、手势状态机 | 纯 Python,无外部依赖 |
| `sigtouch/output` | 鼠标/键盘事件注入 | pynput |
| `sigtouch/ui` | Overlay 覆盖层、系统托盘、设置窗口、调试预览窗 | PySide6 |
| `sigtouch/config` | 配置读写与默认值 | QSettings |
| `sigtouch/app.py` | 入口:装配各模块、生命周期管理、看门狗 | — |

### 数据流

视觉线程每帧产出 `FrameResult`(手部关键点、左右手标签、人脸距离估计、时间戳),经 Qt 信号跨线程发送到主线程。`interaction` 消费 `FrameResult`:更新光标位置、驱动手势状态机,产生的动作交给 `output` 注入;Overlay 用同一份关键点重绘手部轮廓。

**关键设计约束:感知与决策分离。** 手势状态机与光标映射只消费纯数据结构,不接触摄像头与 MediaPipe,因此可用录制的关键点序列做确定性回放测试。

## 4. 核心算法

### 4.1 距离估计

Face Landmarker 输出双眼虹膜中心。按成人平均瞳距 63mm 反推:

```
距离 = 相机焦距(像素) × 63mm ÷ 双瞳像素间距
焦距(像素) = 画面宽度(像素) ÷ (2 × tan(水平视场角/2))
```

相机水平视场角在设置中配置(默认 60°)。距离结果做滑动平均(只驱动慢变化的渲染缩放,精度要求低)。检测不到人脸时保持上次距离值并启动挂起计时。

### 4.2 手部轮廓缩放

设置中配置屏幕物理尺寸(对角线英寸,默认 24)。由距离与屏幕尺寸计算屏幕在用户视野中的张角;张角越小(人远或屏幕相对小),渲染放大系数越大:

手部轮廓大小定义为"占屏幕高度的比例",保持其在用户视野中的张角恒定,需满足 比例 ∝ 距离 ÷ 屏幕物理尺寸,即:

```
scale = clamp((用户距离 / 0.6m) × (24吋 / 屏幕对角线英寸), 0.5, 3.0)
```

基准:0.6m 处 24 吋屏 scale = 1.0。人越远、或屏幕越小,scale 越大。

### 4.3 光标映射

- 取摄像头画面中央"交互框"(四边留白比例可调,默认 15%)线性映射到目标屏幕全域。
- 光标锚点:食指指尖与拇指指尖的中点(捏合动作中该点位移最小)。
- One Euro 滤波平滑(截止频率与速度系数可调)。
- **捏合冻结**:捏合判定成立的瞬间冻结光标位置约 150ms,防止手指闭合动作把光标带偏导致点错。

### 4.4 手势状态机

所有距离阈值按手掌尺寸(手腕到中指根部)归一化,适应远近变化;判定采用滞回(进入/退出双阈值)防抖;每个离散手势有独立冷却时间防连发。

| 手势 | 判定条件 | 动作 |
|---|---|---|
| 拇指+食指快捻 | 指尖距离<进入阈值,且 <250ms 内松开 | 左键单击 |
| 拇指+食指捻住移动 | 捏合保持 >250ms | 拖拽(mouseDown→移动→松开时 mouseUp) |
| 拇指+中指快捻 | 同左键判定 | 右键单击 |
| 三指捻(拇指+食指+中指)保持并上下移动 | 三指尖聚拢 | 滚动,垂直速度映射滚轮量 |
| OK 手势保持 0.5s | 拇指食指成环 **且其余三指伸直**(与捏合的区分特征) | 回车(单次触发) |
| 手掌前推 | 掌心朝向摄像头 且 手部包围盒面积在短窗口内快速增大 | 退格(单次触发) |

状态机状态:`IDLE → PINCH_PENDING → (CLICK | DRAGGING) / SCROLLING / OK_PENDING / PUSH_DETECTED`,任何状态在手部丢失时回到 `IDLE` 并释放已按下的按键。

**防误触总策略**:
- 无人脸(没人看屏幕)持续 N 秒(默认 3s)→ 挂起注入与渲染,恢复检测到即自动继续;
- 手离开交互框或放下 → 光标停住,不产生任何事件;
- OK 与推手触发时,Overlay 在手部旁短暂显示反馈图标(⏎ / ⌫)。

## 5. UI 设计

### 5.1 Overlay 覆盖层

- 无边框全屏窗口:`WA_TranslucentBackground` + `WindowStaysOnTopHint` + `WA_TransparentForMouseEvents`(点击穿透)。
- 渲染:21 关键点连成手掌剪影 + 圆头描边,白色约 35% 不透明度、边缘微光(Oculus 风格);按 4.2 缩放系数放大;手势反馈图标短暂叠加。
- 多显示器:设置中选择投影的目标屏,光标映射与 Overlay 同屏。

### 5.2 系统托盘

- 图标三态:运行中 / 已暂停 / 异常(摄像头丢失、权限缺失)。
- 菜单:启用/暂停切换、打开设置、调试预览(摄像头画面+关键点叠加,调阈值用)、退出。
- 关闭设置窗口不退出应用;退出仅通过托盘菜单。

### 5.3 设置窗口(QSettings 持久化)

| 标签页 | 内容 |
|---|---|
| 摄像头 | 设备选择、分辨率、水平视场角(距离估计校准) |
| 交互 | 灵敏度(交互框边距)、平滑强度、各手势独立开关与阈值、捏合/OK 停留时长 |
| 显示 | 屏幕对角线英寸、轮廓透明度/颜色、目标显示器 |
| 通用 | 开机自启、全局暂停快捷键 |

开机自启实现:Windows 注册表 `Run` 键 / macOS LaunchAgent plist / Linux `~/.config/autostart` .desktop。

## 6. 错误处理

- 摄像头断开:托盘转异常态,后台按退避间隔重试重连,恢复后自动继续。
- 视觉线程看门狗:主线程定时检查帧心跳,超时(>2s 无帧)则销毁并重建视觉管线。
- macOS 权限:首次运行检测摄像头与辅助功能(Accessibility,注入必需)权限,缺失时弹引导对话框跳转系统设置。
- 手部丢失时若处于拖拽/滚动中,立即释放按键,防止"鼠标卡死按下"。

## 7. 平台限制(文档中明示)

- Windows、macOS、Linux(X11):全功能。
- **Linux Wayland:pynput 注入受限,第一版声明为已知限制**(后续版本可加 uinput/ydotool 后端);Overlay 在部分 Wayland 合成器上置顶行为可能受限。

## 8. 测试策略

- `interaction` 层为纯函数核心:用录制/合成的关键点序列做单元测试,覆盖每种手势的触发、误触抑制、冷却、手部丢失释放。
- `perception` 距离估计:合成虹膜坐标验证公式与滤波。
- 光标映射与 One Euro 滤波:确定性输入输出测试。
- 端到端:调试预览窗手动 QA,附三平台检查清单(权限、托盘、Overlay 穿透、注入)。

## 9. 打包与分发

- PyInstaller **onedir** 模式(Qt 动态库独立存在,满足 LGPL 替换要求)。
- 产物:Windows(Inno Setup 安装器)、macOS(.app,代码签名后续处理)、Linux(AppImage)。
- MediaPipe `.task` 模型文件随包捆绑(PyInstaller hook 收集)。

## 10. 性能目标

- 视觉管线 640×480 @ 30fps,端到端(采集→注入)延迟 < 100ms。
- 空闲(挂起态)CPU 占用最小化:挂起时降低采集帧率至 5fps 仅做人脸检测。

## 11. 明确不做(第一版)

- 双手同时交互(仅跟踪一只手,取置信度最高者)。
- 视线定位光标、注视点校准。
- 自定义手势录制。
- Wayland 原生注入后端。

## 12. 实现偏差记录

- 看门狗阈值为 5s(spec §6 原写 2s):实现中 `_check_watchdog` 使用 5000ms 无帧判定卡死,
  权衡挂起态 5fps 采集下的正常帧间隔,避免误触发重建。
- overlay 颜色(`display/overlay_color`)为配置项,但 v1 未在设置界面暴露对应控件,
  仅可通过 QSettings 直接写入调整。
- macOS 启动时仅检测辅助功能(Accessibility)权限;摄像头权限由系统在首次采集时
  自行弹出授权对话框,应用不做额外检测或引导。
